What are Composable functions ?

Pijush Manna
September 19, 2025

Composable functions are Kotlin Functions annoted with @Composable annotation. Is serves as the fundamental UI blocks of Compose. Key characteristics include their immutability, the ability to be called multiple times with the same arguments without side effects, and their role in recomposition, where a new UI is generated when their input parameters change.

Key Concepts :

  • @Composable annotation : It tells the system that this function is used for building UI in Compose.
  • Declarative UI : You describe what the UI should look like rather than manipulate pre-made view components.
  • Input Parameters : Composable function accepts input parameters which decides how the UI should look like.
  • Recomposition : Composable functions are immutable in nature. But they can be redrawn once change in state this phenomenon is called recomposition. Only the part of the UI will be redrawn for which state is changed.
  • State : State refers to any value that can change overtime and triggers recomposition on every change is called a State.

Stateful composable vs Stateless composable :

  • Stateful Composables : Composables that manages their state internally is called Stateful Composables. For example TextField manages its state internally.
  • Stateless Composables: Composables that takes their state from outside. Is called Stateless Composables.For example Text takes its state from outside.
